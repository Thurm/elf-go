## 地图层级设计

### 四层渲染结构

```
┌─────────────────────────────────────┐
│  顶层 (Overlay Layer)                │
│  - 树冠、屋檐、桥梁下方              │
│  - 在角色上方渲染                    │
└─────────────────────────────────────┘
           ↑
┌─────────────────────────────────────┐
│  角色层 (Character Layer)            │
│  - 玩家角色                          │
│  - NPC                               │
│  - 动态物体                          │
└─────────────────────────────────────┘
           ↑
┌─────────────────────────────────────┐
│  中层 (Object Layer)                 │
│  - 建筑、家具                        │
│  - 树木底部、岩石                    │
│  - 装饰物品                          │
└─────────────────────────────────────┘
           ↑
┌─────────────────────────────────────┐
│  底层 (Ground Layer)                 │
│  - 草地、道路、水体                  │
│  - 地毯、地板                        │
│  - 所有基础地面                      │
└─────────────────────────────────────┘
```

### 层级详细说明

#### 1. 底层（Ground Layer）
- **Z-index**: 0
- **内容**:
  - 草地、泥土、沙地等基础地形
  - 道路、水体
  - 室内地板、地毯
- **渲染顺序**: 最先渲染
- **碰撞检测**: 不参与碰撞

#### 2. 中层（Object Layer）
- **Z-index**: 10
- **内容**:
  - 建筑墙壁
  - 家具
  - 树木底部
  - 岩石
  - 栅栏等障碍物
- **渲染顺序**: 底层之后
- **碰撞检测**: 参与碰撞，不可通过的区域在此层标记

#### 3. 角色层（Character Layer）
- **Z-index**: 20-29（根据 Y 坐标动态调整）
- **内容**:
  - 玩家角色
  - NPC
  - 可移动的物体
- **渲染顺序**: Y 坐标小的先渲染（后方的角色先画）
- **碰撞检测**: 参与碰撞

#### 4. 顶层（Overlay Layer）
- **Z-index**: 30
- **内容**:
  - 树冠
  - 屋檐
  - 桥梁下方
  - 门帘等覆盖物
- **渲染顺序**: 最后渲染
- **碰撞检测**: 不参与碰撞（仅视觉效果）

### 碰撞层（Collision Layer）

#### 碰撞数据格式
```javascript
// 碰撞层是一个 2D 数组，与地图尺寸相同
const collisionMap = [
  [0, 0, 1, 1, 0, 0, ...],  // 0 = 可通过, 1 = 不可通过
  // ...
];
```

#### 碰撞类型
| 值 | 类型 | 说明 |
|----|------|------|
| 0 | Free | 可自由通过 |
| 1 | Blocked | 完全阻挡（墙、建筑） |
| 2 | Water | 水体（不可通过） |
| 3 | Portal | 传送点 |
| 4 | Interaction | 交互点（NPC、门） |
| 5 | Grass | 草丛（可通过，可能遇敌） |
